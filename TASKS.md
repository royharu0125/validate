# 程式碼庫問題盤點與建議任務

## 任務 1：修正拼字／命名錯誤（文件）
- **問題**：`README.md` 的「專案結構」區塊寫的是 `validator/` 目錄，但目前專案根目錄實際並沒有這個資料夾，容易讓讀者誤解執行路徑與匯入方式。
- **建議修正**：將結構描述改為實際路徑（例如 `./validator.py`、`./README.md`），並補上正確專案根目錄名稱。
- **完成條件**：新手依 README 操作時，不會再誤以為需要進入不存在的 `validator/` 目錄。

## 任務 2：修正程式錯誤（布林值被視為合法年齡）
- **問題**：`validator.py` 使用 `isinstance(age, int)` 驗證型別；在 Python 中 `bool` 是 `int` 的子類別，`True/False` 會被當作 `1/0`，造成 `True` 可能被誤判為合法年齡。
- **建議修正**：改為拒絕布林值（例如 `type(age) is int` 或 `isinstance(age, int) and not isinstance(age, bool)`）。
- **完成條件**：`age=True` 與 `age=False` 都會被標記為 `age_out_of_range`（或新定義的型別錯誤代碼）。

## 任務 3：修正註解／文件落差
- **問題**：函式文件字串目前將「非整數型別」與「數值超界」都歸類為 `age_out_of_range`，語意上不精確，容易讓維運人員誤以為只有範圍問題。
- **建議修正**：新增更精準的錯誤代碼（例如 `age_invalid_type`）或在 README 明確註記「型別錯誤也使用 `age_out_of_range`」的設計理由。
- **完成條件**：程式行為與文件敘述一致，讀者可直接從錯誤碼理解錯誤類型。

## 任務 4：改進測試
- **問題**：目前僅有 `__main__` 的手動示例，缺乏可重複執行的自動化測試。
- **建議修正**：建立 `pytest` 測試，至少覆蓋：
  1. 合法資料（全數通過）
  2. 缺少 `age`
  3. 邊界值 `min_age` / `max_age`
  4. 非整數（字串、`None`、布林值）
  5. 缺少 `id` 時回傳 `id=None`
- **完成條件**：`pytest` 可一鍵執行，且能穩定重現上述規則。
